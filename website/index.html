claude-email

Control Claude Code Agents via Email

An email-driven wrapper for the Claude Code CLI with an integrated MCP chat relay for managing multiple Claude Code agents. Poll IMAP, verify senders with GPG signatures or a shared secret, execute commands, and reply via SMTP — all from any email client.

View on GitHub

Features

✉

Email Control

Use any email client as a universal remote control for Claude Code. Send a command, get the result back in your inbox.

🔒

Dual Authentication

Verify every incoming command with GPG signatures or a shared secret. Only authorized senders can execute commands.

💬

Multi-Agent Chat

MCP-based chat relay lets multiple Claude Code agents coordinate through a shared SQLite message bus with SSE transport.

⚙

Agent Management

Spawn, message, and manage agents directly from email. Use @agent-name to target specific agents, spawn to create new ones.

🚀

Service Management

User-level systemd services — no sudo required. Auto-restart, dependency ordering, and lingering support out of the box.

✅

Quality

204 tests with 99% code coverage. SQLite WAL mode for safe concurrent access. Python 3.12, fully typed, no shell injection.

How It Works

The system runs as two user-level systemd services. claude-email polls an IMAP mailbox, validates the sender via GPG or shared secret, extracts the command from the email body, and executes it through claude --print. The result is sent back as a threaded email reply.

claude-chat is an MCP SSE server backed by SQLite in WAL mode. It acts as a message bus for multiple Claude Code agents. Agents register, send messages, and poll for responses through MCP tools. The email service acts as the user's avatar on this bus — routing email commands to agents and relaying responses back as emails.

Email Client ↓ IMAP Poll (claude-email) ↓ Sender Verification (GPG / Secret) ↓ ├─ Direct command → claude --print → SMTP reply └─ Chat command → MCP bus (claude-chat) → Agent → SMTP reply

Installation

Clone the repository and run the installer:

git clone https://github.com/cocodedk/claude-email.git
cd claude-email
cp .env.example .env    # edit with your IMAP/SMTP credentials
bash install.sh         # creates venv + systemd services

The installer sets up a Python virtual environment, installs dependencies, and registers both claude-email and claude-chat as user-level systemd services.

# Check service status
systemctl --user status claude-email
systemctl --user status claude-chat

# View logs
journalctl --user -u claude-email -f
