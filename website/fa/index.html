claude-email

کنترل عامل‌های Claude Code از طریق ایمیل

یک پوشش ایمیل‌محور برای خط فرمان Claude Code با رله چت MCP یکپارچه برای مدیریت چندین عامل Claude Code. صندوق ورودی IMAP را بررسی می‌کند، فرستنده را با امضای GPG یا رمز مشترک تایید می‌کند، دستورات را اجرا می‌کند و پاسخ را از طریق SMTP ارسال می‌کند — همه از هر سرویس‌گیرنده ایمیلی.

مشاهده در GitHub

ویژگی‌ها

✉

کنترل از طریق ایمیل

از هر سرویس‌گیرنده ایمیلی به عنوان کنترل از راه دور جهانی برای Claude Code استفاده کنید. دستوری بفرستید و نتیجه را در صندوق ورودی خود دریافت کنید.

🔒

احراز هویت دوگانه

هر دستور ورودی را با امضای GPG یا رمز مشترک تایید کنید. فقط فرستندگان مجاز می‌توانند دستورات را اجرا کنند.

💬

چت چند عاملی

رله چت مبتنی بر MCP به چندین عامل Claude Code اجازه می‌دهد از طریق گذرگاه پیام SQLite مشترک با انتقال SSE هماهنگ شوند.

⚙

مدیریت عامل‌ها

عامل‌ها را مستقیما از ایمیل ایجاد، پیام‌رسانی و مدیریت کنید. از @agent-name برای هدف‌گیری عامل خاص و spawn برای ایجاد عامل جدید استفاده کنید.

🚀

مدیریت سرویس

سرویس‌های systemd سطح کاربر — بدون نیاز به sudo. راه‌اندازی مجدد خودکار، ترتیب وابستگی و پشتیبانی از lingering به صورت پیش‌فرض.

✅

کیفیت

۱۹۳ تست با ۹۹٪ پوشش کد. حالت WAL در SQLite برای دسترسی همزمان ایمن. پایتون ۳.۱۲، کاملا تایپ‌شده، بدون تزریق شل.

نحوه عملکرد

سیستم به صورت دو سرویس systemd سطح کاربر اجرا می‌شود. claude-email صندوق ورودی IMAP را بررسی می‌کند، فرستنده را از طریق GPG یا رمز مشترک تایید می‌کند، دستور را از متن ایمیل استخراج می‌کند و آن را از طریق claude --print اجرا می‌کند. نتیجه به صورت پاسخ ایمیل رشته‌ای بازگردانده می‌شود.

claude-chat یک سرور MCP SSE است که توسط SQLite در حالت WAL پشتیبانی می‌شود. این سرور به عنوان گذرگاه پیام برای چندین عامل Claude Code عمل می‌کند. عامل‌ها ثبت‌نام می‌کنند، پیام ارسال می‌کنند و پاسخ‌ها را از طریق ابزارهای MCP دریافت می‌کنند. سرویس ایمیل به عنوان آواتار کاربر روی این گذرگاه عمل می‌کند — دستورات ایمیلی را به عامل‌ها هدایت می‌کند و پاسخ‌ها را به صورت ایمیل بازمی‌گرداند.

Email Client ↓ IMAP Poll (claude-email) ↓ Sender Verification (GPG / Secret) ↓ ├─ Direct command → claude --print → SMTP reply └─ Chat command → MCP bus (claude-chat) → Agent → SMTP reply

نصب

مخزن را کلون کرده و نصب‌کننده را اجرا کنید:

git clone https://github.com/cocodedk/claude-email.git
cd claude-email
cp .env.example .env    # ویرایش با اطلاعات IMAP/SMTP خود
bash install.sh         # ایجاد venv + سرویس‌های systemd

نصب‌کننده یک محیط مجازی پایتون ایجاد می‌کند، وابستگی‌ها را نصب می‌کند و هر دو سرویس claude-email و claude-chat را به عنوان سرویس‌های systemd سطح کاربر ثبت می‌کند.

# بررسی وضعیت سرویس
systemctl --user status claude-email
systemctl --user status claude-chat

# مشاهده لاگ‌ها
journalctl --user -u claude-email -f
